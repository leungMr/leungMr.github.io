# Open DR: An Approximate Diﬀerentiable Renderer

ECCV ,2014

摘要：逆向图形试图获取传感器数据并推断3D几何、照明、材质和运动，以便图形渲染器能够逼真地再现观察到的场景。然而，渲染器的设计是为了解决图像合成的前向过程。在另一个方向上，我们提出了一种近似的可微分renderer（DR），它明确地模拟了模型参数变化和图像观测之间的关系。我们描述了一个可公开使用的OpenDR框架，该框架可以方便地表示前向图样模型，然后自动获得模型参数的导数并对其进行优化。OpenDR建立在一个新的自动差异化软件包和OpenGL的基础上，提供了一种局部优化方法，可以将其纳入概率编程框架中。通过使用OpenDR解决从Kinect深度和RGB数据**估计人体形状的问题，我们展示了OpenDR编程的强大性和简单性。

# 1 引文

计算机视觉作为一种综合分析方法有着悠久的传统[9,24]，并且仍然是一大类生成方法的核心。在这种自上而下的方法中，视觉被定义为搜索模型的参数，该模型被渲染以生成图像（或图像的特征），然后与图像像素（或特征）进行比较。模型可以采用多种形式的不同真实感，但当模型和渲染过程被设计为生成真实图像时，该过程通常称为逆图形[3,33]。从某种意义上说，这种方法试图对产生世界形象的物理过程进行逆向工程。

我们定义了一个观察函数**f（Θ）**作为依赖于参数Θ的正向渲染过程。最简单的优化将解决参数最小化渲染图像和观测图像强度之间的差异，E（Θ）=||f（Θ）− I ||^2。当然，我们将指定更复杂的函数，包括强大的惩罚和优先级，但是最基本的想法依然保留—尽量减少合成数据和观测数据之间的差异。虽然已经有很多关于这个过程的文章，很多方法都属于这个范畴，但很少有方法真正采用逆图形方法。高维性使优化目标成为挑战；渲染器具有较大的输出空间，而真实渲染器需要较大的输入参数空间。从根本上说，正向渲染功能是复杂的，包含它的优化方法通常是专门构建的，效果非常好。简而言之，图形渲染器通常不是为反转而构建的。

在这里，我们完全接受视觉作为逆向图形的观点，并提出了一个框架，使其更加实用。真实感图形引擎可用于渲染正向过程，并且存在许多区别性方法直接从图像中恢复场景属性。这两种模型都没有明确地对观测值（像素或特征）如何随模型参数平滑变化进行建模。这些导数对于高维问题的优化是必不可少的，并且为每个应用手工构造这些导数是非常繁重的。在这里，我们描述了一个基于不同渲染的通用框架。我们将不同的渲染器（DR）定义为一个过程，（1）提供像素作为模型参数的函数，以模拟物理成像系统，（2）提供像素值相对于这些参数的导数。为了实用，DR也必须快速；这意味着它必须有硬件支持。因此，我们直接使用OpenGL。因为我们将其公开，所以我们称我们的框架为OpenDR(http://open-dr.org).

由于许多方法制定生成模型并对其进行区分，为什么到目前为止还没有通用DR框架？也许是因为渲染似乎是不可微分的。在某种程度上这是正确的，但问题是它在实践中是否重要。**所有渲染器都是近似的，我们的DR也不例外。**我们在第3节和第4节中描述了我们的近似，并论证了在实践中，“近似可微”实际上非常有用。

我们的目标不是渲染，而是反向渲染：我们希望指定并缩小一个目标，其中渲染器只是其中的一部分。为此，我们的DR建立在一个新的Python自动差异化框架（称为Chumpy）的基础上，该框架使编程紧凑且相对容易。我们的公共AutoDiff框架可以轻松扩展OpenDR的基本功能，以解决特定问题。例如，可以将形状空间中的顶点参数化，而不是将输入几何体指定为顶点；或者在输出中，可能需要像素、边或矩的拉普拉斯金字塔，而不是原始像素值。虽然自动微分并没有消除编写这些函数的需要，但它确实消除了手动微分的需要。

利用这一点，我们定义了OpenDR框架，该框架支持计算机视觉中广泛的现实问题。OpenDR框架提供了一种简洁高效的方式来表达计算机视觉问题，而无需担心如何区分这些问题。这是第一个公开提供的区分图像生成过程的框架。

为了评估OpenDR并说明如何使用它，我们提供了两个示例。第一个是一个简单的“hello world”示例，用于说明OpenDR的基本思想。第二个更复杂的示例涉及将3D人体形状的关节和可变形模型与Kinect的图像和范围数据进行匹配。这里我们优化3D身体形状、姿势、照明、反照率和相机参数。这是一个复杂而丰富的生成模型，对其进行优化通常具有挑战性；使用OpenDR，可以直接表达和优化。

逆向图形的理论已经很成熟，但缺少的是从图像中进行渲染和优化之间的直接联系。图形是一种综合。推理是从观察到模型（或参数）的过程。差异渲染通过将观察图像中的变化与模型参数的变化明确关联，以具体的方式将它们联系起来。

# 3 定义我们的流程

设f（Θ）为渲染函数，其中Θ是用于创建图像的所有参数的集合。这里我们将Θ分解为顶点位置V、相机参数C和逐顶点亮度A：因此Θ={V，C，A}。逆图形本质上是近似的，确定正向过程及其差异的近似值是很重要的，我们的远期模型采用以下近似值：

外观（Appearance，A）：逐像素表面外观建模为平滑纹理和逐顶点亮度的乘积，这样亮度结合了反射和照明的效果。球面谐波和点光源作为 OpenDR 的一部分可用；其他直接照明模型易于构造。不明确支持全局照明，包括内部反射和所有复杂的照明效果。

几何（Geometry，V）：我们假设一个 3D 场景由三角形近似，由顶点 V 参数化，可以选择将背景图像（或深度渲染器的深度图像）放置在几何体后面。 对象的数量没有明确的限制，DR 甚至不“知道”它是在渲染一个还是多个对象； 它的货币是三角形，而不是物体。

照相机（Camera，C）：我们通过它们的采样中心值来近似连续像素强度。 我们使用来自 OpenCV 的针孔加失真相机投影模型。 与其他投影相比，它的主要区别在于图像失真模型 [7] 的细节，而后者又源自 [11]。

我们的近似值接近于现代图形管道所做的近似值。一个重要的例外是外观：现代图形管道支持根据用户定义的函数在曲面上按像素分配，而在这里我们支持按顶点用户定义的函数（在两个曲面之间插入颜色）。虽然我们也支持纹理贴图，但我们还不支持纹理贴图上强度值的差异。与de LaGorce[20]不同，我们不支持关于纹理的导数；虽然他们使用双线性插值，但由于我们使用了mipmapping，我们需要三线性插值。这是今后的工作。

我们强调，如果OpenDR被证明是有用的，用户将有希望扩展它，从而放松许多这些假设。这里我们描述最初的版本。

# 4.3 相对于2D图像坐标的强度微分

为了估计∂F∂U、 我们首先将像素分割为遮挡边界像素和内部像素，灵感来源于[20]。边界像素引起的变化主要是由于一个曲面替换为另一个曲面，而内部像素引起的变化与曲面面片的图像空间投影变换有关。通过识别边缘上的像素（a）通过深度测试（由渲染器形成）和（b）将三角形与相反法线连接在一起：一个三角形朝向摄影机，另一个背向摄影机，通过渲染过程可获得边界像素的指定。我们考虑一个像素的三个分类：内部、内部/边界和多个边界。

内部像素：像素不包含遮挡边界。因为外观是插值纹理和插值颜色的产物，所以强度变化相对Interior/boundary于几何体变化是分段平滑的。对于内部像素，我们使用[17]采用的图像空间一阶泰勒展开方法。为了理解这种方法，请考虑在图像空间中向右平移一个像素的补丁：每个像素被它的左邻右舍替换，这类似于Sobel-Lter的应用。重要的是，我们不允许这种过滤交叉或包括边界像素（这种情况[17]未处理，因为遮挡未建模）。

具体地说，在不与遮挡边界相邻的像素上，我们使用核1/2[−1, 0, 1]执行水平滤波，在与遮挡相邻的像素上，我们使用 [0, -1, 1] 进行水平过滤； 对于右侧的像素相邻遮挡边界，我们使用 [−1, 1, 0]； 并且在两侧都有遮挡边界时，我们将导数近似为零。 对于垂直过滤，我们使用相同的内核转置

内部/边界：一个像素与一个遮挡边界相交。 对于内部/边界情况，我们使用具有 kernel12[−1, 0, 1] 及其转置的图像空间过滤。 这近似了一种差异（前景边界与其后面的表面之间的差异）与另一种差异（前景边界与其后面的表面相邻的像素之间的差异）。 我们不是在遮挡边界后面“偷看”，而是使用相邻像素作为代理并假设差异不是太大。 实际上，边界梯度几乎总是比被遮挡的背景表面补丁的梯度大得多，因此在优化过程中占据主导地位。

多个边界：像素中存在多个遮挡边界。虽然对象空间方法以建模所有几何体为代价为这些像素提供精确的导数，但我们将其视为内部/边界情况。之所以这样做，是因为很少有像素会受到这种情况的影响，而且精确的对象空间计算会非常昂贵。

总而言之，差异处理的最重要近似值出现在边界像素上，我们将一个差异（附近像素减去遮挡像素）与另一个差异（附近像素减去几乎遮挡像素）近似。我们发现这在实践中有效，但重要的是要认识到更好的近似是可能的

作为一个实现细节，我们的方法在请求原始渲染图像时需要一个渲染通道，当需要导数时需要额外的三个通道（用于边界识别、三角形识别和重心坐标）。 每次通过都需要从 GPU 回读。

# 4.4 软件基础

灵活性对于不同渲染器的通用性至关重要；自定义功能应易于设计，无需手动进行区分。为此，我们使用自动微分[10]来计算仅给定正向过程特定值的导数，而不采用有限的微分方法。作为OpenDR版本的一部分，我们包含了一个新的自动差异化框架（Chumpy）。这个框架本质上是Numpy[25]，它是Python中的一个数字包，可以区分。通过共享Numpy的大部分API，这允许使用流行API对问题进行正向说明。这反过来允许对不属于渲染器一部分的模型进行正向指定，并允许对渲染器的上层进行最小程度的指定。尽管考虑了其他自动差异化框架[4,35,21]，我们还是将Numpy包装起来以便于使用。我们的整个系统依赖于Numpy[25]、Scipy[16]和OpenCV[7]。

# 6.1 来自Kinect的身体形状

现在，我们使用Kinect 作为输入设备解决身体测量估计问题。在综合分析方法中，必须估计许多参数，以有效地解释图像和深度证据。通过最小化超过一百万个残差的贡献，我们有效地估计了数千个参数（每个顶点的反照率是最大的贡献），这对于无导数的方法是不切实际的。

受试者被要求在两个相隔45度的视图中形成A姿势或T姿势；然后在看不到对象的情况下执行捕获。这将生成三个深度和三种颜色的图像，其中大多数状态（姿势除外）假定在两个视图中保持不变。

—潜在变量：照明参数AL、逐顶点反照率AC、彩色摄影机平移T和身体参数B：因此Θ={AL，AC，T，B}。

—可观测量：深度图像D1...n ，和彩色图像I1...n、 n=3。

# 7 总结

计算机视觉中的许多问题都通过渲染过程中的差异化得到了有效解决。这并不新鲜。新的是，我们提供了一个易于使用的框架，用于渲染器差异和目标公式。这使得在Python中很容易定义正向模型并对其进行优化。我们已经用一个具有挑战性的问题证明了这一点，即从图像和距离数据中估计身体形状。通过使用OpenSourceLicense发布OpenDR（请参阅http://open-dr.org)，我们希望创建一个使用并为这项服务做出贡献的社区。希望通过提供工具使其工作更容易，这将推动视觉逆图形的研究。

差异化渲染有其局限性。当使用GB高斯棱锥之间的差异时，基本问题是重叠：如果模拟和观察对象在棱锥中没有重叠，模拟对象将不会记录朝向观察对象的梯度。可以使用没有重叠限制（例如，力矩）的像素函数来解决这一问题，但基本限制是可见性：真实观察到的特征不会拖拽因轮廓状态而完全被遮挡的模拟特征

因此，差异化渲染只是难题的一部分：我们相信，知情采样[15]和概率图形编程[22]对于逆向渲染的重要应用也是必不可少的。尽管如此，我们希望许多人将从OpenDR平台中受益。

未来的探索可能包括通过整合全局照明提高图像真实感。它还可能包括现代渲染管道的更多功能（例如，通过片段着色器进行差异化）。我们还致力于构建用于后验估计的“可集成渲染器”；虽然可以使用标准采样方法来近似此类积分，但在有限的领域内，可能会有与图形相关的技术以更直接的方式进行集成。
