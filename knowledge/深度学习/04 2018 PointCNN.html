# PointCNN

PointCNN: Convolution On X-Transformed Points ，Neural Information Processing Systems （2018）

原论文链接：https://arxiv.org/abs/1801.07791

原作者实现的TensorFlow代码：https://github.com/yangyanli/PointCNN

# X-transformation 思想

  • 论文的主要创新点

    • 让网络去学一个转置矩阵X，该矩阵能对某个特定顺序的输入进行处理，得到一个与顺序无关的feature。

    • 即输入数据先经过X变换矩阵处理，再执行Convolution的操作。

  • 本质是 k * k 的变换矩阵，k是点云的数量。

  • 其与PointNet中的T-Net的思路类似，都是希望先变换在一定程度上解决无序的问题。

# X-Conv 算法

算法描述：

  1. 将所有邻居点的坐标转化为相对p的坐标P’；
  2. 用一个MLPδ网络，将邻居点的位置信息P’转化为feature信息Fδ（提升维度）；*
  3. 将转化得到的feature Fδ，与邻居点自己的feature F （比如color或者normal）拼接，得到新的feature F*；
  4. 用一个MLP网络计算出特定输入顺序（P’）对应的X矩阵；
  5. 用矩阵X对特定顺序的feature F*矩阵进行处理，得到Fx；（目的是重新排列F*，达到特征与顺序无关的目的）
  6. 执行Convolution，得到Fp的特征（输出）。

# PointCNN 结构

与图像CNN的区别：

  1. 局部邻域的提取方式（K*K大小的patchs vs. 中心点的局部邻域）
  2. 学习局部区域信息的方式（Conv vs. X-Conv）

网络结构：

图：用于分类（a和b）和分割（c）的PointCNN架构，其中 N 和 C 表示每一层输出代表点数量和特征维数，K 是每个代表点的相邻点数量，D 是X-Conv卷积的膨胀率（dilation rate）。

从前一层中选取代表点集的方法，暂时的实现是：

  • 对分类问题：随机下采样。
  • 对语义分割问题：最远点采样。

分类网络，b是对a的改进（增强）：

  • 原因：常规流程，可以把每个点的感受野大小粗略定义为K/N （K是邻域点数目，N是前一层的点数目）。在分类问题中（图a），最后一层（顶层）的X-Conv的点数骤减为1，使得彻底训练顶层X-Conv层的效率很差。

  • 改进：提出更密集的网络连接（图b），保持原网络的深度，增加每一层中代表点的感受野的大小，同时保持临近点数目为K（核大小不变）

  • 做法：未改进时，每个代表点P的特征从N个点中取K个临近点进行聚合；改进后，从N个点中取K*D个点，然后再从中均匀采集K个点做特征聚合。输出（分类结果）也从1变为4，最终结果对4做平均。

分割网络c，Conv-DeConv的结构，遵循U-Net的设计：

  • Conv-DeConv结构：DeConv负责将全局信息传播到逐点
  • Conv 和 DeConv 都是相同的X-Conv算子
  • ELU 是PointCNN中使用的非线性激活函数
  • 算法第6步的Conv，使用分离卷积(Separable convolution )替代传统卷积来减少参数数量和计算量
  • 在最后完全连接的层之前施加Dropout以减少过拟合
  • 使用ADAM优化器，初始学习率为0.01。

# 数据增广

目的：为了提高可推广性

对于X-Conv中的参数的训练，显然，如果相邻点对于特定代表点始终是相同顺序的相同集合，是很不利的。为了提高可推广性，我们提出随机抽样和缩减输入点，使得相邻点集和顺序可能因批次不同而不同。

为了训练以N个点为输入的模型，使用 N(N,(N/8)){\rm N}\left( {N,\left( {N/8} \right)} \right)N(N,(N/8)) 个点进行训练，其中 N{\rm N}N 表示高斯分布。我们发现，这种策略对于PointCNN的成功训练至关重要。
